2.	OBJECTIVE

3.	THEORY
The Client–Server model is a distributed application structure that partitions tasks between the provider of a resource or service (the server) and the service requester (the client).
•	HTTP (Hypertext Transfer Protocol): An application-layer protocol used for transmitting hypermedia documents. It follows a request-response pattern.
•	Request Method: Indicates the desired action to be performed (e.g., GET for retrieving data).
•	Status Codes: Issued by the server to indicate the outcome of the request (e.g., 200 OK for success, 404 Not Found for missing resources).
•	Headers: Key-value pairs that provide essential metadata about the request or the response content.

3.	THEORY
The Node.js File System (fs) module allows you to work with the file system on your computer.
•	fs.writeFile(): Used to asynchronously write data to a file, replacing the file if it already exists.
•	fs.readFile(): Used to read the content of a file. It requires an encoding type (like 'utf8') to return a string instead of a Buffer.
•	fs.open() & fs.close(): Used for more granular control. Opening a file creates a "File Descriptor" (an integer), which must be closed to prevent memory leaks and file locking.
•	File Paths: Node.js uses relative paths (e.g., ./test.txt) or absolute paths to locate files within the workspace.

•	To establish a connection between PHP and a MySQL database using the mysqli extension.
•	To implement an HTML form for data entry.
•	To execute SQL INSERT queries to store data.
•	To execute SQL SELECT queries to fetch and render data in an HTML table.
3.	THEORY
PHP interacts with databases (typically MySQL) to create dynamic, data-driven websites.
•	Connection: The mysqli_connect() function initializes a connection between the script and the database server.
•	Data Insertion: Data sent via the POST method from an HTML form is captured in PHP and sent to the database using an INSERT INTO SQL statement.
•	Data Retrieval: The SELECT statement is used to fetch rows. PHP loops through these rows using mysqli_fetch_assoc() to display them inside <table>, <tr>, and <td> tags.
4.	CODE
A.	Database Setup (SQL)
Before running the PHP script, the following table must exist in the database: CREATE DATABASE lab_db;
USE lab_db;

);

EXPERIMENT 10




To build a multilingual React application using useContext.
2.	Objective
•	To avoid prop drilling
•	To manage global language state
3.	Theory: useContext allows sharing data across components without passing props manually.
4.	Code:
import React, { createContext, useContext, useState } from "react"; const LangContext = createContext(); function App() { const [lang, setLang] = useState("en"); return (
<LangContext.Provider value={lang}>
<button onClick={() => setLang("en")}>English</button>
<button onClick={() => setLang("hi")}>Hindi</button>
<Message />
</LangContext.Provider>
);
}
function Message() { const lang = useContext(LangContext); return <h2>{lang === "en" ? "Hello" : "नमस्ते"}</h2>;
}
export default App;
 
4.	Output

5.	Conclusion
The multilingual app works successfully using useContext.
